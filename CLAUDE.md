## ファイル作成時の注意点（ファイル作成時必ず確認）

- ファイル作成時に、そのファイルがGithubに挙げられるべきではないと判断した場合には、必ず.gitignoreに指定してください。
- 例えば、個人情報が含まれるファイルや、機密情報が含まれるファイルなどは、Githubに挙げるべきではありません。

## Git workflow

- 機能追加・修正を行う前に、必ず新しいブランチを作成すること。
- 作業は現在のブランチに直接行わず、目的ごとにブランチを分けること。
- 作業が完了したら、変更内容をコミットすること。
- コミット後は必ず push すること。
- push 後は必ず Pull Request を作成すること。
- Pull Request には、目的・変更内容・確認内容・残課題を記載すること。
- mainへの直接コミットは禁止。

## Task tracking

- 実装した内容は `README.md` に記載すること。
- これから実装しようとしている内容も `README.md` に記載すること。
- 残タスクは `README.md` に記載すること。

## Before coding

- 実装前に、要求内容を整理し、作業方針を明確にすること。
- 影響範囲が不明な場合は、関連ファイルを確認してから実装すること。
- 不明点を放置したまま実装を進めないこと。
